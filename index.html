HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Inline CSS styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only.
  • Inline CSS styles are useful for quick and permanent changes but are less flexible than external and internal style sheets, because each inline style must be edited individually whenever you make a design change.
  • Internal style sheets contain CSS rules for the webpage in the head section of the HTML file. The rules only apply to that page, and you can configure CSS classes and IDs, which are used to identify elements, to style multiple elements in the page code.
  • External CSS style sheets are the most common way to style a webpage, this is preferred because it allows developers to keep all their CSS rules in a seperate file, which makes design changes easier

Git

  • "cd filename:" Opens folder
  • "cd ..:" Moves you back one folder
  • "code .:" Opens VS Studio
  • "git status:" checks what branch we are currently on
  • "git pull origin main:" pulls the remote main branch into the local main branch so that the local and remote branches are in sync for the time being
  • "git checkout -b branch-name:" creates a new branch and switches to it
  • "git add -A:" adds all files in the working branch
  • "git commit -m 'short description of changes':" -m associates a message with the commit
  • "git push origin branch-name" will push up your changes to the remote repository
